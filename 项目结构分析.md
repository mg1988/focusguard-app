# FocusGuard 应用 - 项目结构分析

## 📋 项目概述

**FocusGuard** 是一款桌面菜单栏专注力管理应用，通过摄像头实时监测用户的面部状态，帮助提升专注力。主要功能包括：
- 面部检测与走神提醒
- 瞌睡检测与抓拍
- 坐姿检测与渐进式提醒
- 专注时间统计与数据分析
- 多语言国际化支持

## 📁 核心模块详解

### 1️⃣ **应用入口** - `focus_macApp.swift`

**功能**：
- 应用主入口点，采用 SwiftUI 的 `@main` 声明
- 配置菜单栏常驻模式（`MenuBarExtra`）
- 初始化 `FocusViewModel` 作为全局状态管理
- 动态菜单栏图标集成

**关键代码**：
- 使用 `MenuBarExtra` 创建菜单栏应用
- 动态图标展示专注进度、坐姿状态
- 支持隐藏文字标签优化系统空间分配

---

### 2️⃣ **数据模型层** - `Models/`

#### `PostureState.swift` - 坐姿状态模型
**功能**：
- 定义 5 种坐姿状态枚举（良好、弯腰、侧倾、太近、太远）
- 提供本地化名称、图标符号、状态颜色
- `PostureStats` 结构体：统计良好/不良坐姿次数、计算良好率

#### `DailyStats.swift` - 每日统计数据
**功能**：
- 记录每日专注时长、走神次数、瞌睡次数
- 支持 Codable 协议用于持久化
- 提供 `empty` 静态变量用于初始化

#### `DistractionSnapshot.swift` - 抓拍快照模型
**功能**：
- 存储走神/瞌睡时抓拍的照片信息
- `SnapshotType` 枚举区分抓拍类型
- 包含时间戳、图片路径、触发时长等元数据

---

### 3️⃣ **业务逻辑层** - `ViewModels/`

#### `FocusViewModel.swift` - 核心 ViewModel
**功能**：
- 应用状态管理（专注状态、时间、计数等）
- 面部检测与坐姿检测数据绑定
- 专注模式启停控制
- 走神/瞌睡判定逻辑
- 抓拍照片管理
- 数据持久化（UserDefaults）
- 渐进式坐姿提醒系统

**核心属性**：
- `status`: 专注状态（idle/active/distracted）
- `focusTime`: 专注时长
- `distractionCount`: 走神次数
- `drowsyCount`: 瞌睡次数
- `currentPosture`: 当前坐姿
- `snapshots`: 抓拍照片数组
- `history`: 历史统计数据

**关键方法**：
- `toggleFocusMode()`: 切换专注模式
- `handleFaceDetectionUpdate()`: 处理面部检测
- `handlePostureUpdate()`: 处理坐姿更新
- `takeDistractionSnapshot()`: 走神抓拍
- `startBadPostureTimer()`: 渐进式坐姿提醒计时器
- `saveDailyStats()`: 保存每日数据

---

### 4️⃣ **服务层** - `Services/`

#### `FaceDetectionManager.swift` - 面部检测服务
**功能**：
- 摄像头权限管理与采集
- Vision 框架集成进行面部地标检测
- EAR（Eye Aspect Ratio）算法检测闭眼
- 坐姿检测逻辑（距离、低头、侧倾）
- 抓拍照片生成与保存

**核心技术**：
- `AVCaptureSession`: 视频采集
- `VNDetectFaceLandmarksRequest`: 面部地标识别
- 滑动窗口平滑处理（避免误判）
- 连续帧验证机制

**坐姿检测算法**：
1. **距离检测**：基于面部 bounding box 大小
2. **低头检测**：鼻子与嘴巴的垂直距离
3. **侧倾检测**：双眼水平高度差

#### `NotificationManager.swift` - 通知服务
**功能**：
- 系统通知权限管理
- 发送走神/瞌睡/坐姿提醒通知
- 渐进式提醒系统（3 级强度）
- 声音与触感反馈

**提醒级别**：
- **Level 1**（3 秒）：温和提醒（短促单音 + 轻微震动）
- **Level 2**（10 秒）：中等提醒（双音提示 + 明显震动）
- **Level 3**（20 秒）：强烈提醒（连续三音 + 连续震动）

## 🔧 技术栈与架构

### 架构模式
**MVVM (Model-View-ViewModel)**
- **Model**: `PostureState`, `DailyStats`, `DistractionSnapshot`
- **View**: `MainView`, `SettingsView`, `StatisticsView` 等
- **ViewModel**: `FocusViewModel` 统一管理状态与业务逻辑

### 核心技术
- **SwiftUI**: 声明式 UI 框架
- **Vision**: 面部地标检测
- **AVFoundation**: 摄像头采集
- **Combine**: 响应式数据绑定
- **UserNotifications**: 系统通知
- **ServiceManagement**: 开机自启动

### 设计亮点
1. **渐进式提醒系统**：3 级强度提醒，避免过度打扰
2. **平滑算法**：滑动窗口 + 连续帧验证，减少误判
3. **隐私保护**：100% 本地处理，无云端上传
4. **国际化**：6 种语言支持
5. **数据持久化**：UserDefaults 存储历史数据与设置

## 🎯 核心功能流程

### 1. 专注模式启动流程
```
用户点击"开启专注" → FocusViewModel.toggleFocusMode()
  → 状态设为.active
  → FaceDetectionManager.startDetection()
  → 启动主计时器
  → 菜单栏图标开始更新
```

### 2. 走神检测流程
```
摄像头检测不到面部 → distractionTimer 开始计时
  → 超过阈值（灵敏度决定）
  → distractionCount++
  → 抓拍照片
  → NotificationManager 发送提醒（声音 + 震动 + 通知）
```

### 3. 坐姿提醒流程
```
检测到不良坐姿 → badPostureTimer 启动
  → 3 秒：Level 1 温和提醒
  → 10 秒：Level 2 中等提醒
  → 20 秒：Level 3 强烈提醒
  → 菜单栏图标闪烁
```

## 📝 总结

FocusGuard 是一款架构清晰、功能完善的桌面专注力管理工具。采用 MVVM 架构实现关注点分离，利用 Vision 和 AVFoundation 实现精准的 facial detection，通过渐进式提醒系统提供人性化的用户体验，同时严格保护用户隐私。代码结构模块化，便于后续功能扩展与维护。
