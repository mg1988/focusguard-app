# GitHub Pages 404 错误排查指南

## 常见原因和解决方案

### 原因 1：Pages 功能未启用 ⭐ 最常见

**检查步骤**：

1. 访问：https://github.com/mg1988/focusguard-app/settings/pages
2. 查看 "Source" 部分
3. 应该设置为：
   - **Deploy from a branch**
   - **Branch**: `main` 
   - **Folder**: `/ (root)`

**解决方法**：
```
1. 如果没有启用，点击 "Deploy from a branch"
2. Branch 选择 "main"
3. Folder 选择 "/ (root)"
4. 点击 "Save"
```

---

### 原因 2：文件不在根目录

**检查步骤**：

1. 访问：https://github.com/mg1988/focusguard-app
2. 查看文件列表
3. 确认 `privacy.html` 在根目录（不是在子文件夹中）

**解决方法**：
```bash
# 如果文件在子文件夹中，需要移动到根目录
cd /Users/genweimi/Desktop/flutter/ios/focus/focus_mac/AppStoreAssets

# 检查文件
ls -la *.html

# 重新推送
git add .
git commit -m "Fix: move files to root"
git push
```

---

### 原因 3：分支名称不对

**检查步骤**：

1. 访问：https://github.com/mg1988/focusguard-app/branches
2. 查看默认分支是 `main` 还是 `master`

**解决方法**：

如果是 `master`：
```bash
# 在 Pages 设置中选择 master 分支
# 或者重命名分支
git branch -m main
git push -u origin main
```

---

### 原因 4：部署还未完成

**检查步骤**：

1. 访问：https://github.com/mg1988/focusguard-app/actions
2. 查看最新的 GitHub Pages 部署状态
3. 应该是绿色的 ✅

**解决方法**：
```
等待 2-5 分钟，GitHub Pages 需要时间构建
```

---

### 原因 5：文件名大小写问题

**检查**：

- GitHub 是区分大小写的
- 确保 URL 中的文件名与实际文件名完全一致

**正确**：
```
privacy.html (全小写) ✅
```

**错误**：
```
Privacy.html (大写 P) ❌
Privacy.HTML (全大写) ❌
```
