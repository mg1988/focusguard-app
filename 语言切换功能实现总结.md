### 3. 设置页面语言选择器 (SettingsView)
**文件位置**: `/focus_mac/Views/SettingsView.swift`

**新增 UI 组件**:
```swift
SettingsSection(title: NSLocalizedString("language_settings", comment: "语言设置")) {
    SettingsRow(icon: "globe", color: .purple, title: NSLocalizedString("language_settings", comment: "")) {
        Picker("", selection: $viewModel.selectedLanguage) {
            ForEach(AppLanguage.allCases) { language in
                Text(language.localizedName).tag(language)
            }
        }
        .pickerStyle(.menu)
        .frame(width: 150)
        .onChange(of: viewModel.selectedLanguage) { newLanguage in
            viewModel.changeLanguage(newLanguage)
        }
    }
    .subtitle(NSLocalizedString("language_select_desc", comment: "Select your preferred language"))
}
```

**功能**:
- ✅ 使用 macOS 原生 Picker 组件（菜单样式）
- ✅ 显示所有可用语言的本地化名称
- ✅ 实时切换，立即生效
- ✅ 优雅的 UI 设计，与其他设置项风格统一

## 🔧 技术实现细节

### 1. Bundle 扩展
```swift
extension Bundle {
    private static var bundleLanguage: String?
    
    static func setLanguage(_ language: String?) {
        // 设置关联对象，存储自定义 bundle
    }
    
    override open func localizedString(...) -> String {
        // 重写本地化方法，优先使用自定义 bundle
    }
}
```

**原理**:
- 利用 objc_setAssociatedObject 将语言 bundle 关联到主 bundle
- 重写 `localizedString(forKey:value:table:)` 方法
- 根据用户选择的语言返回对应的翻译

### 2. 通知机制
```swift
NotificationCenter.default.post(
    name: NSNotification.Name("LanguageDidChange"), 
    object: nil
)
```

**作用**:
- 语言切换后发送通知
- UI 组件监听通知并刷新
- 无需手动重启应用

### 3. 数据持久化
```swift
UserDefaults.standard.set(language.rawValue, forKey: "selectedLanguage")
```

**特点**:
- 自动保存用户选择
- 下次启动自动加载
- 与系统设置独立

## ⚠️ 注意事项

### 1. 阿拉伯语 RTL 支持
阿拉伯语是从右到左 (RTL) 语言，可能需要额外的布局调整：

**建议添加**:
```xml
<!-- Info.plist -->
<key>NSApplicationIsRightToLeft</key>
<true/>
```

### 2. 长文本适配
某些语言的翻译可能较长（如德语），需要确保 UI 组件有足够的空间：

**测试重点**:
- 设置页面的行高
- 下拉菜单的宽度
- 按钮和标签的文字截断

### 3. 字体支持
确保所有语言使用的字体都支持对应字符集：
- 中文、日文、韩文：需要 CJK 字体
- 阿拉伯语：需要阿拉伯字体
- 俄语：需要西里尔字母支持
